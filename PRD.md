# PRD - 测测你的人性弱点 (Weak)

> 本文档定义「测测你的人性弱点」产品的完整产品需求，基于卡内基《人性的弱点》设计。
> 每次产品改动前，先更新本文档，再同步修改代码。

---

## 一、产品概述

### 1.1 产品定位

**一句话描述：** 一款基于卡内基《人性的弱点》的AI性格测试工具，通过15道选择题诊断你在人际交往中的"致命弱点"。

**核心用户画像：**
- 主要用户：18-35岁职场年轻人（刚入职/转行/想提升情商）
- 次要用户：对人际交往困惑的学生、自由职业者
- 用户特征：想知道自己在社交中的问题、愿意为"自我认知"付费

### 1.2 核心价值主张

> "15道题，测出你在人际交往中的致命弱点"

用户购买的不是一个标签，而是"知道自己哪里做错了"和"如何改正"的具体路径。

### 1.3 灵感来源：卡内基《人性的弱点》

卡内基把人际交往的弱点归纳为**六大维度**：

| 维度 | 卡内基原文核心 | 测试问法 |
|------|--------------|---------|
| **批评型** | "不要批评、指责或抱怨" | 你在关系中是否倾向于挑剔和批评他人？ |
| **冷漠型** | "真诚地关注他人" | 你是否只在乎自己的需求，忽视对方的感受？ |
| **自恋型** | "谈论对方感兴趣的话题" | 聊天时你是否只说自己，不问对方的事？ |
| **记不住型** | "记住对方的名字" | 你是否经常忘记别人的名字和基本信息？ |
| **防御型** | "在争论中获胜的唯一方法是避免争论" | 遇到分歧时你是否必须赢？ |
| **索取型** | "激发他人内在需求" | 你是否只关注自己想从对方那里得到什么？ |

### 1.4 商业模式

```
小红书发布内容 → 引流到测试网站
        ↓
用户访问 → 输入周码（¥0.99/周码）
        ↓
做15道题 → 获得弱点诊断报告
        ↓
分享到小红书 → 带来二次传播
```

## 二、用户旅程

### 2.1 完整用户旅程

```
步骤1: 在小红书看到"你在人际交往中的致命弱点是什么"
  ↓
步骤2: 被"弱点"概念吸引（每个人都有好奇心想知道自己的缺点）
  ↓
步骤3: 看到精美首页，输入周码
  ↓
步骤4: 做15道选择题（每道题映射一个真实场景）
  ↓
步骤5: 等待AI分析（3秒动画）
  ↓
步骤6: 看到结果（主要弱点 + 弱点解析 + 改变建议）
  ↓
步骤7: 觉得"太准了" / "被戳中了"
  ↓
步骤8: 截图分享到小红书
  ↓
步骤9: 朋友也去做 → 形成传播
```

### 2.2 用户情绪曲线

```
震惊 ↑                        结果页（被戳中）
     |                      ↗
     |                  ↗
     |              ↗
     |           ↗
     |        ↗
     |     ↗
     |  ↗
     +------------------------------→ 时间
     首页   做题中   等待中   结果页
```

**设计原则：**
- 首页：制造"想知道自己弱点"的好奇心
- 做题：每个场景都是日常人际痛点，做完就共鸣
- 结果：必须戳中、有共鸣、可分享

## 三、功能规格

### 3.1 核心功能

| 功能 | 优先级 | 状态 | 说明 |
|------|--------|------|------|
| 首页周码输入 | P0 | ⏳ 待开发 | 输入周码进入测试 |
| 15道选择题 | P0 | ⏳ 待开发 | 场景化题目，6维度分析 |
| AI弱点诊断 | P0 | ⏳ 待开发 | 识别主要弱点 + 次要弱点 |
| 可分享结果 | P0 | ⏳ 待开发 | 截图 + 文案分享 |
| 管理后台 | P1 | ⏳ 待开发 | 码管理 + 统计数据 |
| 小红书自动发货 | P2 | ⏳ 待开发 | 千帆API对接 |

### 3.2 页面规格

#### 3.2.1 首页（/）

**视觉要求：**
- 简洁但有冲击力，第一眼抓住"想认识自己"的欲望
- 主标题："测测你的人性弱点"
- 副标题："15道题，诊断你在人际交往中的致命短板"
- 通行码输入框（核心入口）
- 背景：暗色调 + 人影轮廓（隐喻"看见自己的阴暗面"）

**配色：** 深色系（#1a1a2e 深蓝黑），搭配红色警示感（#e94560）

#### 3.2.2 做题页（/test）

**视觉要求：**
- 每个问题都是一个真实人际场景
- 选项模拟真实对话/行为选择
- 进度条显示当前位置（1/15）

**交互要求：**
- 点击选项 → 下一题（无确认按钮）
- 最后一题 → 自动提交
- 提交后 → 跳转等待页

#### 3.2.3 等待页（/loading）

**视觉要求：**
- 3秒分析动画
- 随机文案（如："卡内基用了20年总结出的智慧..."）
- 品牌元素展示

#### 3.2.4 结果页（/result）

**核心展示：**
- 第一屏：**主要弱点名称**（大字，冲击感，如"你是批评型人格"）
- 第二屏：弱点解析（为什么会这样 + 日常表现）
- 第三屏：改变建议（卡内基的方法论 + 具体行动）
- 第四屏：次要弱点（次要问题）
- 底部：分享按钮

**分享文案：**
```
😱 我在【测测你的人性弱点】中测出了我的致命短板...

我的弱点是：[弱点名称]
为什么： [一句话解析]

卡内基说这是人际关系的最大杀手
快来测测你的 ↓
[链接]
```

#### 3.2.5 管理后台（/admin）

**功能：**
- 查看统计数据（总/今日/本周）
- 更换周码
- 查看最近测试记录

## 四、题库设计

### 4.1 六大维度

| 维度 | 卡内基原则 | 题数 | 高分特征 |
|------|-----------|------|---------|
| 批评型 | 不要批评、指责或抱怨 | 3题 | 习惯性挑剔、总看到别人缺点 |
| 冷漠型 | 真诚地关注他人 | 2题 | 只在乎自己，忽视对方感受 |
| 自恋型 | 谈论对方感兴趣的话题 | 2题 | 聊天只说自己，不问对方 |
| 记不住型 | 记住对方的名字 | 2题 | 经常忘记别人名字/信息 |
| 防御型 | 在争论中必须赢 | 3题 | 遇到分歧就争辩，无法接受不同意见 |
| 索取型 | 激发他人内在需求 | 3题 | 只关注自己想从对方那里得到什么 |

**总计：15道题**

### 4.2 题目示例（场景化设计）

```
场景1：你和同事一起完成项目
同事方案有明显漏洞，你会？
A. 当面指出，让他马上改（批评型 +3）
B. 私底下委婉提醒（正常）
C. 无所谓，反正不是我负责（冷漠型）
D. 表面同意，背后跟别人吐槽（防御型）

场景2：朋友聊天时一直在说自己的事
你会？
A. 打断他，把话题引到自己身上（自恋型）
B. 耐心听，但心里已经走神（冷漠型）
C. 趁机提问，让他说更多（正常）
D. 找个借口先走开（冷漠型）
```

## 六、竞品分析

### 6.1 现有竞品

| 产品 | 优点 | 缺点 |
|------|------|------|
| 各种性格测试 | 话题传播强 | 无深度、无变现 |
| 九型人格测试 | 维度清晰 | 题多、无AI包装 |

### 6.2 Weak的差异化

1. **卡内基IP背书**：全球销量过亿的人际交往经典，有天然信任感
2. **"弱点"比"优点"更吸引人**：测自己的缺点比测优点更有传播力
3. **AI个性化报告**：每人的弱点组合不同，报告千人千面
4. **周码模式**：复用AISoul成熟的变现体系
